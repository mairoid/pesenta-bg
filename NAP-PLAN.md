# НАП план — pesenta.bg като електронен магазин (алтернативен режим)

Източници: [Изисквания към е-търговците (НАП)](https://nra.bg/wps/portal/nra/fiskalni-ustroystva-supto-i-e-magazini/page.turgovia-v-internet-i-e-magazini/page-Iziskvania-kum-e-turgovciete),
[Алтернативен режим (НАП)](https://nra.bg/wps/portal/nra/fiskalni-ustroystva-supto-i-e-magazini/page.turgovia-v-internet-i-e-magazini/page.lternativen-rejim-za-registrirane-i-otchitane-na-prodajbite),
Приложение №33 към чл. 52м, ал. 1 от Наредба Н-18. Съставен: 08.07.2026.

**Правна рамка накратко:** докато сайтът приема само банков превод — регистрация не е
задължителна (но е възможна доброволно). Щом тръгнат картовите плащания през Stripe без
касов апарат → алтернативен режим по чл. 3, ал. 17: регистрация + е-бележка + месечен
одиторски файл. При подаването НАП присвоява **уникален номер на е-магазина**
(чл. 52т, ал. 1) — идва в електронното съобщение-потвърждение.

- [ ] **Домейн pesenta.bg** — регистрация през Register.BG акредитиран регистратор
  (изисква идентификация: КЕП или нотариално заверено заявление). Собственик: БАУФИЗИК ЕООД.
- [ ] **Сайтът онлайн** — GitHub repo + Pages + DNS (стъпките са в README.md).
- [ ] **FormSubmit активация** — първата заявка праща потвърждаващ имейл; клик и готово.
- [ ] **Счетоводителят да потвърди**: (а) становище на НАП № 60-30-36/12.01.2026 за Stripe;
  (б) подава ли се одиторски файл за месец без продажби; (в) как се попълва т. 4.2.5
  (БД на софтуера), при положение че сайтът е статичен и поръчките пристигат по имейл.

## Фаза 1 — Stripe акаунт (преди НАП, защото т. 7 иска Merchant ID)

- [ ] Регистрация на Stripe акаунт на името на „БАУФИЗИК“ ЕООД (подписва управителят);
  верификация на фирмата и банковата сметка.
- [ ] Запиши: **Merchant ID** (acct_…), номер/идентификатор на виртуалния ПОС,

- [ ] Създай 8-те Payment Links + промо кодовете (стъпките са в README → „Активиране на Stripe“).

## Фаза 2 — Регистрация в НАП (преди първото картово плащане)

- [ ] Портал за е-услуги на НАП → услуга **„Информация, подавана от лице по чл. 3 от
  Наредба №Н-18/2006 г., извършващи продажба на стоки/услуги чрез електронен магазин“** (с КЕП).
- [ ] Попълване по Приложение №33 — черновата по-долу.
- [ ] Получаване на **уникалния номер на е-магазина** (в имейла-потвърждение) → запази го;
  влиза в е-бележката, QR кода и одиторския файл.
- [ ] Проверка, че магазинът се появява в Публичния списък на е-магазините на НАП.

## Фаза 3 — Техника по чл. 52о и чл. 52с (строи се от Claude, след НАП номера)

- [ ] **Е-бележка** (документ за регистриране на продажбата) при всяко картово плащане,
  с реквизитите по чл. 52о, ал. 1: наименование/номер/дата на документа; данни на търговеца;
  уникален номер на клиентска поръчка (PSN-…); референтен номер на финансовата трансакция
  (от Stripe); стока/услуга, код на данъчна група, количество, ед. цена, обща сума, начин на
  плащане; **QR код по Приложение №18а**. Изпраща се на клиента по електронен път.
- [ ] **Генератор на одиторския файл** по Приложение №38: скрипт, който от Stripe данните
  за месеца произвежда валиден XML (по официалната XSD схема от портала на НАП),
  с уникалния номер на магазина. Проверка: логическа и аритметична коректност
  (порталът валидира при подаване).
- [ ] Вграждане на уникалния номер на е-магазина в конфигурацията на сайта.
- [ ] Тест: пробна продажба → е-бележка → пробен XML.
